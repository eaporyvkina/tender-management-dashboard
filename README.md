# 🏗️ Tender Management Dashboard  
**Управление строительными проектами (УрФО) | Power BI**

Дашборд для руководства строительной компании, позволяющий контролировать портфель из 40+ проектов, бюджеты, сроки и загрузку менеджеров.

---

## 🎯 Бизнес-задача

Компания управляет проектами в области теплосетей, водоснабжения, котельных и насосных станций в 4 регионах УрФО.  
Руководству не хватало единого окна для:
- мониторинга статусов и бюджетов;
- выявления проблемных проектов;
- оценки загрузки менеджеров;
- анализа рисков по срокам и финансам.

**Цель:** создать интерактивный дашборд, который позволит принимать решения на основе данных за 30 секунд.

---

## 📊 Архитектура данных

### Схема «Звезда»
- **Факт-таблица:** `fact_tenders` (проекты с датами, бюджетами, статусами)
- **Измерения:** `dim_manager`, `dim_status`, `dim_priority`, `dim_project_type`, `dim_region`, `Dates`
- **Вспомогательная таблица:** `Budget_by_Month` (равномерное распределение бюджета по месяцам для корректного тренда)

> *Почему это важно:* звездная схема обеспечивает быструю фильтрацию, простоту DAX и масштабируемость.

## 📈 Ключевые метрики и визуализации

### 1. Executive Summary (главная страница)
- **KPI:** общее количество проектов, бюджет, проблемные проекты, освоение бюджета, рейтинг подрядчиков, рост год-к-году.
- **Динамика активных проектов** – линейный график, показывающий изменение загрузки по месяцам.
- **Распределение по типам и регионам** – горизонтальные гистограммы.
- **Загрузка менеджеров** – таблица с условными гистограммами (data bars).
- **Топ‑3 проблемных проекта** – список с иконками и причинами.

### 2. Финансовый анализ
- **Bullet Chart** – сравнение текущих затрат с планом и прогнозом по каждому проекту.
- **Мера `Monthly Budget`** – сумма распределённого бюджета по месяцам для тренда.

### 3. Матрица рисков (точечная диаграмма)
- **Ось X:** отклонение бюджета (%)
- **Ось Y:** дней до дедлайна
- **Размер точки:** бюджет проекта
- **Цвет:** статус (проблемные выделены красным)

*Инсайт:* проекты в правом верхнем квадранте требуют немедленного вмешательства – они одновременно с перерасходом и близким дедлайном.

## 🧠 Ключевые DAX-меры

```dax
// Накопленный итог проектов (для тренда)
Cumulative Projects = 
CALCULATE(
    COUNTROWS('fact_tenders'),
    FILTER(
        ALL('fact_tenders'),
        'fact_tenders'[Start_Date] <= MAX('Dates'[Date])
    )
)

// Активные проекты в каждый месяц
Active Projects Count = 
CALCULATE(
    COUNTROWS('fact_tenders'),
    FILTER(
        'fact_tenders',
        'fact_tenders'[Start_Date] <= MAX('Dates'[Date]) &&
        'fact_tenders'[Deadline_Date] >= MIN('Dates'[Date])
    )
)

// Матрица рисков – отклонение бюджета
Budget Deviation % = 
DIVIDE(
    SUM('fact_tenders'[Budget_Current]) - SUM('fact_tenders'[Budget_Planned]),
    SUM('fact_tenders'[Budget_Planned])
)

// Вес проблемы для ранжирования
Problem Score = 
VAR DaysLeft = DATEDIFF(TODAY(), SELECTEDVALUE('fact_tenders'[Deadline_Date]), DAY)
VAR Progress = SELECTEDVALUE('fact_tenders'[Progress_Percent])
VAR BudgetDev = [Budget Deviation %]
RETURN
    IF([Is Problem Project] = 1,
        (IF(DaysLeft < 0, 100, 0) +
         IF(DaysLeft < 7, 50, 0) +
         IF(Progress < 30, 30, 0) +
         BudgetDev * 100),
        0
    )
